## 一句话简介

基于《星空》Creation Kit 独立完成的恐怖探索向支线关卡；玩家登上一艘失联的豪华星际邮轮，通过恢复供电、搜寻钥匙与电池、穿越通风管道，逐步揭开事故真相并在货舱完成最终战斗。

## 项目概览

职责｜关卡设计 / 环境叙事 / Gameplay 实现
开发｜8 周 / 单人 / Scrum
结构｜6 个功能区 / 约 16 分钟目标流程
玩法｜探索 / 解谜 / 潜行 / 战斗
工具｜Starfield Creation Kit

我将现实邮轮的功能分区转化为适合第三人称 RPG 的关卡结构，并利用供电前后状态变化、上层奢华空间与下层工业空间的反差、钥匙与电池驱动的循环探索，构建从无力感、调查、威胁显现到最终对抗的完整节奏。

任务阶段｜玩家行动｜体验目标
建立无力感｜在黑暗、失重的主甲板登船｜限制信息与行动，建立未知威胁
恢复秩序｜进入舰桥启动备用电源｜用重力、灯光与门锁变化提供阶段性释放
调查真相｜搜索客舱、船员区与厨房｜通过功能分区和环境痕迹拼合事件经过
威胁升级｜经通风管进入活动区，收集三块电池｜压缩空间、引入敌人并迫使玩家管理资源
最终对抗｜用电池解锁货舱，取得燃料并迎战 Terrormorph｜将前段积累的压力释放为战斗高潮

![关卡总览：六个功能区围绕供电、钥匙与电池目标逐步开放](images/level-overview.jpg)

## 设计目标

1. 不依赖持续剥夺玩家能力，而是通过“有限反制手段”维持恐惧与主动性之间的平衡。
2. 让邮轮的每个功能区同时承担空间辨识、叙事信息和玩法节奏，不把环境叙事做成额外收集项。
3. 通过状态变化、单向捷径和区域重用，让有限规模的室内空间产生持续变化。

## 01｜用空间状态变化建立恐怖节奏

关卡以黑暗、失重的主甲板开场。玩家只能依靠手电辨认空间，并被锁门引导至舰桥。启动备用电源后，重力、照明与部分门锁同时恢复：熟悉的空间由不可控转为可导航，玩家获得短暂安全感，也由被动求生转入主动调查。

供电不是一次单纯的任务交互，而是整个关卡的节奏转折点。它同时改变移动方式、可见范围、空间情绪与可进入路线，让玩家明确感受到自己的行动重塑了环境。

![舰桥：备用电源将灯光、重力和门锁状态整合为一次可感知的空间转变](images/bridge-power.jpg)

设计目标｜用一次交互完成节奏转折，而非只更新任务文字
实现方法｜失重与黑暗开场 / 舰桥作为可见目标 / 灯光、重力、门锁同步变化
设计结果｜玩家在获得方向感与行动能力后，自然进入更开放的调查阶段

## 02｜把现实邮轮转化为可读的游戏空间

我参考现实邮轮的甲板与功能布局，将关卡压缩为舰桥、客舱、船员区、厨房、活动区和货舱六个辨识度明确的区域。上层采用开阔、装饰性强的豪华空间；下层使用紧凑、重复、工业化的服务空间。材质、尺度、照明和陈设共同帮助玩家判断自己身处何处。

真实建筑以容纳人流和满足功能为主，游戏关卡则必须服务镜头、敌人寻路、视线与节奏。因此我放大关键走廊和房间尺度，压缩重复舱室，并用主路径、侧房间和地标控制探索密度。

![乘客区域：大窗、红毯与装饰照明建立豪华上层的视觉身份](images/passenger-luxury.jpg)

![船员区域：会议桌将空旷房间分割为环形路线，并形成稳定的空间锚点](images/crew-loop.jpg)

现实参考｜邮轮功能分区、客舱排列、上下层服务关系
设计转译｜减少楼层与重复房间，强化六个区域的主题差异
玩法调整｜扩大第三人称镜头与大型敌人所需空间，加入回环、遮挡与捷径

## 03｜用循环路线提高空间利用率

关卡并非沿直线逐室推进。玩家从甲板进入舰桥恢复供电，再回到甲板进入客舱；随后通过船员区解锁厨房，经通风管抵达活动区，收集电池后沿大楼梯单向返回客舱，最终解锁货舱。

这条路线让玩家多次回到已经认识的区域，但每次返回时目标、开放通路和威胁状态都不同。通风管提供狭窄的单向过渡，大楼梯则把玩家快速送回主路线，使“重访”承担导航确认、节奏切换和进度反馈，而不是无意义折返。

![乘客大厅：连接客舱、厨房与活动区，是多次重访时确认方向的核心空间](images/passenger-hub.jpg)

首次经过｜建立区域关系，发现被锁路线
再次经过｜供电后开放新通路，进入调查阶段
捷径返回｜从活动区单向回到客舱，带着电池前往货舱

## 04｜让环境叙事服务探索与决策

叙事信息被放置在玩家为了推进任务必须经过的空间中：客舱中的尸体与钥匙提示船员逃亡方向，厨房和通风系统说明幸存者如何绕过封锁，活动区的战斗痕迹与电池将事故规模转化为可操作目标。玩家不是停下来阅读完整背景，而是在寻找路线和资源时逐步拼合邮轮覆灭的过程。

环境叙事也承担导航功能。尸体、破坏痕迹、异常照明和房间状态既是故事证据，也是指向下一个交互点的视觉线索，使叙事与关卡引导共用同一套空间语言。

![餐厅与活动区域：明亮的公共空间被战斗痕迹和资源目标重新定义为威胁区域](images/event-area.jpg)

叙事载体｜尸体 / 终端记录 / 战斗痕迹 / 异常照明 / 空间破坏
关卡作用｜指向钥匙、通风管、电池与后续路线
玩家理解｜从“邮轮为何失联”逐步转向“威胁仍在船上”

## 05｜在恐惧与玩家主动性之间取得平衡

早期测试中，完全没有武器虽然符合恐怖主题，却与《星空》玩家已经建立的战斗预期冲突。玩家面对威胁时只能逃跑，挫败感超过了恐惧感。

我因此在前段提供电磁武器：它能够短暂麻痹敌人，却不能直接消灭主要威胁。玩家获得了观察、脱身与重新规划路线的手段，但仍无法通过正面输出解除风险。进入活动区后，战斗资源与敌人逐步增加，最终在货舱转为完整 Boss 战，实现从回避威胁到正面对抗的压力释放。

阶段｜玩家能力｜威胁与空间｜设计意图
开场探索｜手电与基础移动｜黑暗 / 失重 / 未知空间｜建立脆弱感与悬念
中段调查｜电磁武器 / 钥匙 / 通风管｜狭窄路线 / 潜伏敌人｜给予有限控制权但保留风险
活动区｜常规武器 / 电池目标｜Xenogrub 群体遭遇｜提高资源压力与行动密度
货舱终战｜完整战斗能力｜Xenogrub + Terrormorph｜释放前段积累的压力

![工业下层：更紧凑的服务空间与复杂遮挡，为后段威胁升级建立视觉与空间反差](images/industrial-lower-deck.jpg)

## Gameplay 与任务实现

我使用 Creation Kit 完成六个区域的空间搭建、灯光与重力状态、门锁与钥匙、终端交互、通风管路线、三电池目标、敌人配置、任务阶段和存档点。任务状态将供电、区域开放与最终战斗串联起来，确保玩家每次完成目标后都能从环境变化中获得反馈。

实现模块｜具体内容
空间状态｜失重 / 灯光 / 备用电源 / 门锁 / 单向捷径
任务推进｜钥匙 / 三块电池 / 目标更新 / 货舱解锁
遭遇设计｜Xenogrub / Terrormorph / 敌人数量与触发位置
风险控制｜Boss 房外安全屋 / 存档点 / 大型敌人寻路空间

## 测试迭代

测试发现｜设计判断｜修改｜验证结果
完全没有武器使玩家只能逃跑，挫败感超过恐惧感｜恐怖应来自威胁与资源限制，而非彻底剥夺玩家选择｜在前段提供只能短暂麻痹敌人的电磁武器｜玩家获得脱身手段，但仍无法直接消灭主要威胁
船员区空旷且缺少明确动线｜大房间没有遮挡、地标和路线选择，难以形成探索节奏｜用中央会议桌分割空间，外圈布置走廊与独立房间｜形成可辨认的环形路线，并提高视线与转角变化
最终 Boss 战存档点位于战斗房间内部，可能造成反复载入即死亡｜存档位置没有为玩家提供重新准备的安全状态｜将存档点移至货舱外的安全屋，并调整任务触发位置｜玩家可在进入战斗前补给并重新制定策略，避免死亡循环
大型敌人在狭窄室内移动受阻｜现实比例走廊与《星空》的大型 AI 尺寸不匹配｜扩大关键门洞与货舱战斗空间，减少阻挡物｜改善 Terrormorph 寻路，并为最终战提供可周旋路线

## 项目总结

项目成果（What Went Well）
• 独立完成一条包含六个功能区、状态变化、循环探索、解谜与最终战斗的恐怖向支线关卡。
• 通过豪华上层与工业下层的视觉反差、空间状态变化及环境痕迹建立清晰的区域身份与恐怖节奏。
• 将现实邮轮结构压缩并转译为适合第三人称镜头、任务推进和敌人寻路的可玩空间。

优化方向（Even Better If）
• 环境叙事证据的层次仍可增加，使玩家不仅理解“发生了袭击”，也能更准确地还原事件时间线。
• 最终货舱仍应在白盒阶段进行更多大型敌人专项测试，以进一步丰富掩体、周旋路线与阶段变化。
• 部分区域的尺度先按现实参考搭建，后期才为镜头和 AI 放大；后续应更早建立可复用的游戏尺度标准。

项目收获（What I Learned）
• 现实建筑不能直接复制成游戏关卡。有效的转译需要保留功能关系与空间身份，同时重新设计尺度、路线、视线和内容密度。恐怖节奏也不等于持续压制玩家；通过有限反制、短暂安全感与环境状态变化，玩家的主动选择反而能让威胁更可信。
